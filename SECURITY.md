# Security Policy

## Supported Versions

Security updates are provided only for the latest released version of this
project.

| Version | Supported |
| ------- | --------- |
| Latest  | ✅ Yes    |
| Older  | ❌ No     |

> Please make sure you are running the most recent version before reporting
> a vulnerability.

---

## Reporting a Vulnerability

If you discover a security vulnerability, **do not open a public GitHub issue**.

Instead, please report it privately using one of the following methods:

- **GitHub**: Contact the project maintainer directly  
  https://github.com/Ozymandias089

When reporting a vulnerability, please include:

- A clear description of the issue
- Steps to reproduce the vulnerability
- Potential impact (e.g., data exposure, privilege escalation)
- Any relevant logs, screenshots, or proof-of-concept code

---

## Response and Disclosure

- Reports will be acknowledged as soon as reasonably possible.
- Valid vulnerabilities will be investigated and addressed promptly.
- Once a fix is available, the reporter may be credited (if desired).
- Public disclosure should be coordinated with the maintainer to avoid
  unnecessary risk to users.

The project maintainer is committed to handling all reports responsibly and
with respect for the reporter’s privacy.

---

## Out of Scope

The following are generally considered out of scope:

- Denial of service attacks requiring excessive traffic
- Vulnerabilities in third-party dependencies without a known exploit
- Social engineering attacks
- Issues requiring physical access to infrastructure

---

Thank you for helping keep this project and its users secure.
